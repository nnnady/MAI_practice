# Практика на предприятии АО НТЦ «Модуль»

## Задание: доработка драйвера АПУ-ЦВМ12Р

Доработка Python-драйвера `apupp.py` для пульта **АПУ-ЦВМ12Р**: добавление поддержки контекстного менеджера, рефакторинг конструктора/деструктора и тестирование на реальном устройстве.

---

### 1. Поддержка контекстного менеджера

В классе `Apupp` реализовать поддержку контекстного менеджера:

- при **входе** в контекстный менеджер — устанавливать соединение;
- при **выходе** — разрывать соединение.

### 2. Рефакторинг конструктора и деструктора

- Убрать из конструктора класса `Apupp` установку соединения с АПУ-ЦВМ12Р (отправку команды).
- Удалить деструктор класса `Apupp`.

### 3. Тестовый скрипт

Создать собственный Python-скрипт со следующим функционалом:

- [ ] Установить протокол взаимодействия **APUPP** — импортировать модуль `set_apupp` и вызвать его функцию `run()`.
- [ ] Создать контекстный менеджер класса `Apupp`.
- [ ] Внутри контекстного менеджера установить значение **КПУ** для **ВК1**, равное `0000`.
- [ ] Попробовать устанавливать другие значения, делая паузы между установками — чтобы на пульте было видно переключение светодиодов КПУ.

> **Примечание:** ВК — вычислительный канал ЦВМ12Р, выбор ВК осуществляется аргументом `svk`.

### 4. Запуск на стенде

Запустить скрипт на реальном устройстве.

### 5. Проверка чтением

- Добавить в скрипт проверку установки КПУ командой считывания значения.
- Снова запустить скрипт на реальном стенде.

---
